# ADR-001: Границы сервисов и владение данными

- Статус: принято
- Дата: 2026-07-12

## Контекст

TeamOS должен позволять изменять, развертывать и масштабировать нагруженный домен независимо от остальных. При этом продукт начинает команда из 1–2 разработчиков, поэтому сервис на каждый фронтенд-модуль создаст больше распределённых транзакций и операционной нагрузки, чем пользы.

## Решение

Система состоит из BFF `gateway` и пяти доменных сервисов:

| Сервис | Владелец данных и доменов |
|---|---|
| `company` | auth, компания, пользователи, оргструктура, графики, распределение сделок |
| `kb` | разделы, статьи, версии, поиск, ознакомления |
| `tasks` | доски, колонки, задачи, комментарии, метки |
| `academy` | курсы, уроки, тесты, назначения, прогресс |
| `notifications` | уведомления и SSE-доставка |
| `gateway` | данных не хранит; JWT, CORS, rate limit, маршрутизация |

Каждый сервис является отдельным Go-модулем, образом/бинарником и deploy unit. Он имеет собственные миграции и конфигурацию. На старте все базы размещены в одном PostgreSQL 16, но являются отдельными базами `teamos_<service>`. Чужую базу сервис не читает, межсервисные UUID не имеют внешних ключей.

Общий код допускается только в версионируемом модуле `pkg`. Доменные DTO и модели не разделяются. Импорт чужого `internal` запрещён компилятором и архитектурным правилом. Синхронные связи сведены к контрактам gRPC, остальные связи идут через версионированные события NATS.

Сервисы stateless: восстанавливаемое состояние находится в PostgreSQL/NATS. Адреса БД, шины и соседей задаются только переменными окружения с префиксом сервиса.

## Последствия

- `company` остаётся крупнее остальных, зато пользователь, credential, session, invite и оргструктура меняются транзакционно.
- Перенос сервиса на отдельный сервер требует смены DNS/env и при необходимости логической репликации только его базы.
- Межсервисные выборки и каскады требуют событийной денормализации и допускают короткую eventual consistency.
- Новый домен добавляется новым сервисом без изменения существующих доменных сервисов; меняются gateway-контракт и каталог событий.
